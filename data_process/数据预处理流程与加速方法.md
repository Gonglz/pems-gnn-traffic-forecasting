# 数据预处理流程与加速方法详解

本项目的数据预处理流程分为多个阶段，涵盖原始数据整理、元数据与外部特征处理、掩码生成、插值填补、拓扑构建等。每一步均有对应的加速实现，部分步骤支持 GPU 并行、分块处理、向量化等高效手段。以下详细梳理各步骤及其加速方法，并给出关键代码说明。

---

## 1. 原始数据整理

### 1.1 文件重命名
- **脚本**：`step0_rename.py`
- **功能**：将原始 txt 文件按首行时间戳重命名，便于后续批量处理。
- **加速方法**：批量遍历目录，跳过隐藏/系统文件，直接读取首行，无需全文件扫描。

### 1.2 元数据清洗与合并
- **脚本**：`step01_meta.py`
- **功能**：清洗 D07 探测器元数据、TopoMap 信息，并按 station_id 合并。
- **加速方法**：Pandas 向量化处理，字段裁剪，避免冗余数据。

### 1.3 天气数据拉取与插值
- **脚本**：`step01_weather.py`
- **功能**：基于站点坐标生成网格，调用 Meteostat API 拉取历史天气，重采样到 5 分钟粒度。
- **加速方法**：KDTree 空间索引加速站点-网格映射，Pandas 向量化插值。

### 1.4 原始流量数据长表化
- **脚本**：`step10_dataExtract.py`
- **功能**：批量读取原始 txt，整理为标准长表格式。
- **加速方法**：分块读取，Pandas 向量化处理。

---

## 2. 健康度与掩码生成

### 2.1 日健康因子统计
- **脚本**：`step20_healthCheck.py`、`step20_healthWeigh_gpu.py`
- **功能**：统计每站点每日健康度，支持 GPU 加速。
- **加速方法**：NumPy 向量化、可选 GPU 并行（见 `step20_healthWeigh_gpu.py`）。

### 2.2 健康度填充
- **脚本**：`step21_fillHealth.py`
- **功能**：将健康度表合并到主数据，缺失填默认值。
- **加速方法**：分块处理，tqdm 进度条。

### 2.3 逻辑掩码自动调优
- **脚本**：`step30_logicAutoMask.py`
- **功能**：两阶段参数调优，自动生成逻辑掩码。
- **加速方法**：Joblib 多核并行，分块抽样，向量化 F1-score 计算。

### 2.4 Mahalanobis 掩码
- **脚本**：`step32_MDmask.py`、`step330_tuneMDmask_gpu.py`
- **功能**：分组计算均值/协方差，CUDA kernel 并行计算距离，自适应阈值。
- **加速方法**：Numba @cuda.jit GPU 并行，分组处理。

### 2.5 掩码合并
- **脚本**：`step34_maskMix.py`
- **功能**：三类掩码合并，输出 CSV/Parquet。
- **加速方法**：Pandas 向量化 merge，分块处理。

---

## 3. 插值填补流水线

### 3.1 CPU 版本
- **脚本**：`step40_fillCPU.py`
- **功能**：合并掩码，按 Local/Global/Temporal 顺序插值。
- **加速方法**：KDTree 空间索引，Pandas/NumPy 向量化。

### 3.2 GPU 版本
- **脚本**：`step40_fill_gpu.py`、`step40_fill_fast.py`、`step40_fill_fastest.py`
- **功能**：分块读取，掩码合并，三步插值全部在 GPU 上完成。
- **加速方法**：
  - `cupy`/`cudf`/`numba.cuda` 向量化与并行
  - KDTree/NearestNeighbors GPU 实现
  - Dask 分布式分块处理
  - CUDA kernel 自定义插值算子

### 3.3 Dask 多卡分布式
- **脚本**：`step50_Fill.py`
- **功能**：Dask 多 GPU 分布式插值，支持大规模数据。
- **加速方法**：Dask 集群，分块重分区，CUDA kernel 并行。

## 4. 拓扑结构重建

### 4.1 邻接边生成
- **脚本**：`step52_buildTopo.py`
- **功能**：基于插值后站点，KNN 生成邻接边，输出 PyTorch 格式。
- **加速方法**：scikit-learn NearestNeighbors 向量化。

- `step33_checkMD.py`、`step34_checkMask.py`：掩码检查与可视化

- `step60_trainingData.py`、`step61_build_subgraphs_fast.py`、`step62_precompute_neighbors.py`、`step63_build_data_list.py`：训练数据与子图生成

## 代码加速方法详细说明与代码级别讲解

### 1. 向量化（Pandas/NumPy/cupy/cudf）
**原理**：利用底层 C/Fortran 实现的批量运算，避免 Python for 循环，极大提升数据处理速度。

**典型代码**：
```python
# Pandas 向量化处理
df['timestamp'] = pd.to_datetime(df['timestamp'], format="%m/%d/%Y %H:%M:%S")
df['flow'] = df['flow'].fillna(0)

# cupy/cudf GPU 向量化
import cupy as cp
import cudf
gdf = cudf.DataFrame.from_pandas(df)
gdf['speed'] = cp.sqrt(gdf['flow'] / gdf['occupancy'])
```

### 2. 分块处理（Chunking）
**原理**：将大表分块读写，单次只处理部分数据，降低内存压力，适合超大规模数据。

**典型代码**：
```python
# Pandas 分块读取
reader = pd.read_csv(RAW_LONG, chunksize=5_000_000)
for chunk in reader:
  # 对 chunk 进行处理
  ...
```

### 3. 多核/多卡并行
**原理**：利用多核 CPU（如 joblib、multiprocessing）、多 GPU（如 Dask、numba.cuda）并行处理，显著缩短运行时间。

**典型代码**：
```python
# Joblib 多核并行
from joblib import Parallel, delayed
results = Parallel(n_jobs=8)(delayed(func)(x) for x in data_list)

# Dask 多 GPU 分布式
from dask_cuda import LocalCUDACluster
from dask.distributed import Client
cluster = LocalCUDACluster()
client = Client(cluster)
# dask_cudf 分布式 DataFrame
import dask_cudf
ddf = dask_cudf.read_csv('large.csv')
ddf = ddf.map_partitions(process_func)
```

### 4. 空间索引（KDTree/NearestNeighbors）
**原理**：高效查找空间邻域，常用于地理空间最近邻、插值等。

**典型代码**：
```python
from sklearn.neighbors import KDTree
tree = KDTree(coords, leaf_size=40)
dist, ind = tree.query(query_points, k=5)
```

### 5. 自定义 CUDA kernel
**原理**：用 numba @cuda.jit 编写 GPU 并行核函数，极大加速核心计算（如插值、距离计算等）。

**典型代码**：
```python
from numba import cuda, float32
@cuda.jit
def local_kernel(feat, nbr, mask, out, K):
  i = cuda.grid(1)
  if i < feat.size:
    if mask[i]:
      s = float32(0.0)
      for j in range(K):
        s += feat[nbr[i, j]]
      out[i] = s / K
    else:
      out[i] = feat[i]
```

### 6. 高效 I/O（Parquet/CSV/Arrow）
**原理**：采用高效的列式存储格式（如 Parquet、Arrow），支持分块、并行读写，适合大数据量。

**典型代码**：
```python
# Pandas 写 Parquet
df.to_parquet('output.parquet', engine='pyarrow')
# 读取 Parquet
df = pd.read_parquet('output.parquet')
```
